## execute

The `temporal workflow execute` command starts a new [Workflow Execution](/workflows#workflow-execution) <span id="i-921bc7c9-aee1-4279-aaaf-492d2a263f38" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-921bc7c9-aee1-4279-aaaf-492d2a263f38" class="preview-modal"><div class="modal-header"><div id="x-921bc7c9-aee1-4279-aaaf-492d2a263f38" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#workflow-execution">/workflows#workflow-execution</a></div><div class="preview-modal-title">What is a Workflow Execution?</div><div class="preview-modal-description">A Temporal Workflow Execution is a durable, scalable, reliable, and reactive function execution. It is the main unit of execution of a Temporal Application.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div> and prints its progress.
The command doesn't finish until the Workflow Execution completes.

To execute a [Workflow](/workflows#) <span id="i-67277181-d549-49d7-91b1-b9383518cb6c" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-67277181-d549-49d7-91b1-b9383518cb6c" class="preview-modal"><div class="modal-header"><div id="x-67277181-d549-49d7-91b1-b9383518cb6c" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#">/workflows#</a></div><div class="preview-modal-title">What is a Workflow?</div><div class="preview-modal-description">In day-to-day conversations, the term "Workflow" frequently denotes either a Workflow Type, a Workflow Definition, or a Workflow Execution.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div> from the CLI:
`temporal workflow execute --workflow-id=meaningful-business-id --type=MyWorkflow --task-queue=MyTaskQueue`

Single quotes('') are used to wrap input as JSON.

`temporal workflow execute --workflow-id=meaningful-business-id --type-MyWorkflow --task-queue-MyTaskQueue --input='{"JSON": "Input"}'`

## reset

The `temporal workflow reset` command resets a [Workflow Execution](/workflows#workflow-execution) <span id="i-ca513772-9d01-47d0-906e-ce57073510c3" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-ca513772-9d01-47d0-906e-ce57073510c3" class="preview-modal"><div class="modal-header"><div id="x-ca513772-9d01-47d0-906e-ce57073510c3" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#workflow-execution">/workflows#workflow-execution</a></div><div class="preview-modal-title">What is a Workflow Execution?</div><div class="preview-modal-description">A Temporal Workflow Execution is a durable, scalable, reliable, and reactive function execution. It is the main unit of execution of a Temporal Application.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div>.
A reset allows the Workflow to be resumed from a certain point without losing your parameters or [Event History](/workflows#event-history) <span id="i-2211fe2f-332e-4c2f-a2ec-be2b276849d0" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-2211fe2f-332e-4c2f-a2ec-be2b276849d0" class="preview-modal"><div class="modal-header"><div id="x-2211fe2f-332e-4c2f-a2ec-be2b276849d0" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#event-history">/workflows#event-history</a></div><div class="preview-modal-title">What is an Event History?</div><div class="preview-modal-description">An append-only log of Events that represents the full state a Workflow Execution.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div>.

The Workflow Execution can be set to a given [Event Type](/workflows#event) <span id="i-42000712-ffeb-4aeb-a29e-9ab651184520" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-42000712-ffeb-4aeb-a29e-9ab651184520" class="preview-modal"><div class="modal-header"><div id="x-42000712-ffeb-4aeb-a29e-9ab651184520" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#event">/workflows#event</a></div><div class="preview-modal-title">What is an Event?</div><div class="preview-modal-description">Events are created by the Temporal Cluster in response to external occurrences and Commands generated by a Workflow Execution.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div>.
`temporal workflow reset --workflow-id=meaningful-business-id --type=LastContinuedAsNew`

## reset-batch

The `temporal workflow reset-batch` command resets a batch of [Workflow Executions](/workflows#workflow-execution) <span id="i-6c4fe5e3-f765-469d-b213-7f972c292e0c" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-6c4fe5e3-f765-469d-b213-7f972c292e0c" class="preview-modal"><div class="modal-header"><div id="x-6c4fe5e3-f765-469d-b213-7f972c292e0c" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#workflow-execution">/workflows#workflow-execution</a></div><div class="preview-modal-title">What is a Workflow Execution?</div><div class="preview-modal-description">A Temporal Workflow Execution is a durable, scalable, reliable, and reactive function execution. It is the main unit of execution of a Temporal Application.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div> by `resetType`.
Resetting a [Workflow](/workflows#) <span id="i-98df91dd-81ac-4b37-9c92-16c9b60376ec" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-98df91dd-81ac-4b37-9c92-16c9b60376ec" class="preview-modal"><div class="modal-header"><div id="x-98df91dd-81ac-4b37-9c92-16c9b60376ec" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#">/workflows#</a></div><div class="preview-modal-title">What is a Workflow?</div><div class="preview-modal-description">In day-to-day conversations, the term "Workflow" frequently denotes either a Workflow Type, a Workflow Definition, or a Workflow Execution.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div> allows the process to resume from a certain point without losing your parameters or [Event History](/workflows#event-history) <span id="i-ea2e96ba-221f-44c3-b6ee-ba56a84cd9f9" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-ea2e96ba-221f-44c3-b6ee-ba56a84cd9f9" class="preview-modal"><div class="modal-header"><div id="x-ea2e96ba-221f-44c3-b6ee-ba56a84cd9f9" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#event-history">/workflows#event-history</a></div><div class="preview-modal-title">What is an Event History?</div><div class="preview-modal-description">An append-only log of Events that represents the full state a Workflow Execution.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div>.

The input file must have a [Workflow ID](/workflows#workflow-id) <span id="i-65c0c6bb-ae6e-47a9-a85d-1920ed6d90b5" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-65c0c6bb-ae6e-47a9-a85d-1920ed6d90b5" class="preview-modal"><div class="modal-header"><div id="x-65c0c6bb-ae6e-47a9-a85d-1920ed6d90b5" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#workflow-id">/workflows#workflow-id</a></div><div class="preview-modal-title">What is a Workflow Id?</div><div class="preview-modal-description">A Workflow Id is a customizable, application-level identifier for a Workflow Execution that is unique to an Open Workflow Execution within a Namespace.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div> on each line.

## terminate

The `temporal workflow terminate` command terminates a [Workflow Execution](/workflows#workflow-execution) <span id="i-252a199b-9208-4ee3-9258-089588d21f62" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-252a199b-9208-4ee3-9258-089588d21f62" class="preview-modal"><div class="modal-header"><div id="x-252a199b-9208-4ee3-9258-089588d21f62" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#workflow-execution">/workflows#workflow-execution</a></div><div class="preview-modal-title">What is a Workflow Execution?</div><div class="preview-modal-description">A Temporal Workflow Execution is a durable, scalable, reliable, and reactive function execution. It is the main unit of execution of a Temporal Application.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div>

Terminating a running Workflow Execution records a [`WorkflowExecutionTerminated` event](/references/events#workflowexecutionterminated) <span id="i-7a2163c4-a7ae-4386-8819-3d89c33a366d" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-7a2163c4-a7ae-4386-8819-3d89c33a366d" class="preview-modal"><div class="modal-header"><div id="x-7a2163c4-a7ae-4386-8819-3d89c33a366d" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/references/events#workflowexecutionterminated">/references/events#workflowexecutionterminated</a></div><div class="preview-modal-title">Events reference</div><div class="preview-modal-description">Events are created by the Temporal Cluster in response to external occurrences and Commands generated by a Workflow Execution.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/reference">reference</a></div></div> as the closing Event in the [Event History](/workflows#event-history) <span id="i-22ee1696-1490-4892-a0b8-cee1b8dee60c" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-22ee1696-1490-4892-a0b8-cee1b8dee60c" class="preview-modal"><div class="modal-header"><div id="x-22ee1696-1490-4892-a0b8-cee1b8dee60c" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#event-history">/workflows#event-history</a></div><div class="preview-modal-title">What is an Event History?</div><div class="preview-modal-description">An append-only log of Events that represents the full state a Workflow Execution.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div>.
Any further [Command](/workflows#command) <span id="i-ab89ddf0-b1c7-41c5-9059-6ab430719067" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-ab89ddf0-b1c7-41c5-9059-6ab430719067" class="preview-modal"><div class="modal-header"><div id="x-ab89ddf0-b1c7-41c5-9059-6ab430719067" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#command">/workflows#command</a></div><div class="preview-modal-title">What is a Command?</div><div class="preview-modal-description">A Command is a requested action issued by a Worker to the Temporal Cluster after a Workflow Task Execution completes.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div> Tasks cannot be scheduled after running this command.

Workflow terminations require a valid [Workflow ID](/workflows#workflow-id) <span id="i-fca09442-9d31-4cc9-8385-7221bae37317" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-fca09442-9d31-4cc9-8385-7221bae37317" class="preview-modal"><div class="modal-header"><div id="x-fca09442-9d31-4cc9-8385-7221bae37317" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#workflow-id">/workflows#workflow-id</a></div><div class="preview-modal-title">What is a Workflow Id?</div><div class="preview-modal-description">A Workflow Id is a customizable, application-level identifier for a Workflow Execution that is unique to an Open Workflow Execution within a Namespace.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div> to function.

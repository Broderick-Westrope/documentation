## execute

The `temporal workflow execute` command starts a new [Workflow Execution](/workflows#workflow-execution) <span id="i-4b9c1196-bb04-4aba-a54b-8cdd74807e79" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-4b9c1196-bb04-4aba-a54b-8cdd74807e79" class="preview-modal"><div class="modal-header"><div id="x-4b9c1196-bb04-4aba-a54b-8cdd74807e79" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#workflow-execution">/workflows#workflow-execution</a></div><div class="preview-modal-title">What is a Workflow Execution?</div><div class="preview-modal-description">A Temporal Workflow Execution is a durable, scalable, reliable, and reactive function execution. It is the main unit of execution of a Temporal Application.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div> and prints its progress.
The command doesn't finish until the Workflow Execution completes.

To execute a [Workflow](/workflows#) <span id="i-678d02f3-fb39-46d2-90f9-29677b688495" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-678d02f3-fb39-46d2-90f9-29677b688495" class="preview-modal"><div class="modal-header"><div id="x-678d02f3-fb39-46d2-90f9-29677b688495" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#">/workflows#</a></div><div class="preview-modal-title">What is a Workflow?</div><div class="preview-modal-description">In day-to-day conversations, the term "Workflow" frequently denotes either a Workflow Type, a Workflow Definition, or a Workflow Execution.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div> from the CLI:
`temporal workflow execute --workflow-id=meaningful-business-id --type=MyWorkflow --task-queue=MyTaskQueue`

Single quotes('') are used to wrap input as JSON.

`temporal workflow execute --workflow-id=meaningful-business-id --type-MyWorkflow --task-queue-MyTaskQueue --input='{"JSON": "Input"}'`

## reset

The `temporal workflow reset` command resets a [Workflow Execution](/workflows#workflow-execution) <span id="i-c5677d35-b881-4923-9859-f155e177b1c3" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-c5677d35-b881-4923-9859-f155e177b1c3" class="preview-modal"><div class="modal-header"><div id="x-c5677d35-b881-4923-9859-f155e177b1c3" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#workflow-execution">/workflows#workflow-execution</a></div><div class="preview-modal-title">What is a Workflow Execution?</div><div class="preview-modal-description">A Temporal Workflow Execution is a durable, scalable, reliable, and reactive function execution. It is the main unit of execution of a Temporal Application.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div>.
A reset allows the Workflow to be resumed from a certain point without losing your parameters or [Event History](/workflows#event-history) <span id="i-28b7a4ed-9008-4490-8a34-a058af619c7f" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-28b7a4ed-9008-4490-8a34-a058af619c7f" class="preview-modal"><div class="modal-header"><div id="x-28b7a4ed-9008-4490-8a34-a058af619c7f" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#event-history">/workflows#event-history</a></div><div class="preview-modal-title">What is an Event History?</div><div class="preview-modal-description">An append-only log of Events that represents the full state a Workflow Execution.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div>.

The Workflow Execution can be set to a given [Event Type](/workflows#event) <span id="i-3d0970f7-a1b2-4e44-ad31-68f8bae3d721" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-3d0970f7-a1b2-4e44-ad31-68f8bae3d721" class="preview-modal"><div class="modal-header"><div id="x-3d0970f7-a1b2-4e44-ad31-68f8bae3d721" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#event">/workflows#event</a></div><div class="preview-modal-title">What is an Event?</div><div class="preview-modal-description">Events are created by the Temporal Cluster in response to external occurrences and Commands generated by a Workflow Execution.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div>.
`temporal workflow reset --workflow-id=meaningful-business-id --type=LastContinuedAsNew`

## reset-batch

The `temporal workflow reset-batch` command resets a batch of [Workflow Executions](/workflows#workflow-execution) <span id="i-152a212b-f75a-4c9d-bb9c-c39d1677b71c" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-152a212b-f75a-4c9d-bb9c-c39d1677b71c" class="preview-modal"><div class="modal-header"><div id="x-152a212b-f75a-4c9d-bb9c-c39d1677b71c" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#workflow-execution">/workflows#workflow-execution</a></div><div class="preview-modal-title">What is a Workflow Execution?</div><div class="preview-modal-description">A Temporal Workflow Execution is a durable, scalable, reliable, and reactive function execution. It is the main unit of execution of a Temporal Application.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div> by `resetType`.
Resetting a [Workflow](/workflows#) <span id="i-d92826f8-f5ef-4da9-a570-c4bb4e1856c6" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-d92826f8-f5ef-4da9-a570-c4bb4e1856c6" class="preview-modal"><div class="modal-header"><div id="x-d92826f8-f5ef-4da9-a570-c4bb4e1856c6" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#">/workflows#</a></div><div class="preview-modal-title">What is a Workflow?</div><div class="preview-modal-description">In day-to-day conversations, the term "Workflow" frequently denotes either a Workflow Type, a Workflow Definition, or a Workflow Execution.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div> allows the process to resume from a certain point without losing your parameters or [Event History](/workflows#event-history) <span id="i-c02362f1-b17c-4700-a899-4ef9cb2b806e" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-c02362f1-b17c-4700-a899-4ef9cb2b806e" class="preview-modal"><div class="modal-header"><div id="x-c02362f1-b17c-4700-a899-4ef9cb2b806e" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#event-history">/workflows#event-history</a></div><div class="preview-modal-title">What is an Event History?</div><div class="preview-modal-description">An append-only log of Events that represents the full state a Workflow Execution.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div>.

The input file must have a [Workflow ID](/workflows#workflow-id) <span id="i-8e6b550c-63fe-4b9b-a781-4907c62df818" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-8e6b550c-63fe-4b9b-a781-4907c62df818" class="preview-modal"><div class="modal-header"><div id="x-8e6b550c-63fe-4b9b-a781-4907c62df818" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#workflow-id">/workflows#workflow-id</a></div><div class="preview-modal-title">What is a Workflow Id?</div><div class="preview-modal-description">A Workflow Id is a customizable, application-level identifier for a Workflow Execution that is unique to an Open Workflow Execution within a Namespace.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div> on each line.

## terminate

The `temporal workflow terminate` command terminates a [Workflow Execution](/workflows#workflow-execution) <span id="i-9fa93cb3-f088-47d6-9efa-455e608b80f4" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-9fa93cb3-f088-47d6-9efa-455e608b80f4" class="preview-modal"><div class="modal-header"><div id="x-9fa93cb3-f088-47d6-9efa-455e608b80f4" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#workflow-execution">/workflows#workflow-execution</a></div><div class="preview-modal-title">What is a Workflow Execution?</div><div class="preview-modal-description">A Temporal Workflow Execution is a durable, scalable, reliable, and reactive function execution. It is the main unit of execution of a Temporal Application.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div>

Terminating a running Workflow Execution records a [`WorkflowExecutionTerminated` event](/references/events#workflowexecutionterminated) <span id="i-63b65c23-8467-4eb9-9310-ba0dacf81595" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-63b65c23-8467-4eb9-9310-ba0dacf81595" class="preview-modal"><div class="modal-header"><div id="x-63b65c23-8467-4eb9-9310-ba0dacf81595" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/references/events#workflowexecutionterminated">/references/events#workflowexecutionterminated</a></div><div class="preview-modal-title">Events reference</div><div class="preview-modal-description">Events are created by the Temporal Cluster in response to external occurrences and Commands generated by a Workflow Execution.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/reference">reference</a></div></div> as the closing Event in the [Event History](/workflows#event-history) <span id="i-ba1826af-c5a3-49b3-a773-94f5b230c8bf" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-ba1826af-c5a3-49b3-a773-94f5b230c8bf" class="preview-modal"><div class="modal-header"><div id="x-ba1826af-c5a3-49b3-a773-94f5b230c8bf" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#event-history">/workflows#event-history</a></div><div class="preview-modal-title">What is an Event History?</div><div class="preview-modal-description">An append-only log of Events that represents the full state a Workflow Execution.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div>.
Any further [Command](/workflows#command) <span id="i-57565a05-77b7-49d3-9ace-b81754b6d66e" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-57565a05-77b7-49d3-9ace-b81754b6d66e" class="preview-modal"><div class="modal-header"><div id="x-57565a05-77b7-49d3-9ace-b81754b6d66e" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#command">/workflows#command</a></div><div class="preview-modal-title">What is a Command?</div><div class="preview-modal-description">A Command is a requested action issued by a Worker to the Temporal Cluster after a Workflow Task Execution completes.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div> Tasks cannot be scheduled after running this command.

Workflow terminations require a valid [Workflow ID](/workflows#workflow-id) <span id="i-492e3f1a-9ffd-4a4c-ad6c-f2a879d8ac34" class="clickable-i clickable-link-preview">i</span><div id="preview-modal-492e3f1a-9ffd-4a4c-ad6c-f2a879d8ac34" class="preview-modal"><div class="modal-header"><div id="x-492e3f1a-9ffd-4a4c-ad6c-f2a879d8ac34" class="clickable-x clickable-link-preview">x</div><b>Link preview:</b>&nbsp;&nbsp<a href="/workflows#workflow-id">/workflows#workflow-id</a></div><div class="preview-modal-title">What is a Workflow Id?</div><div class="preview-modal-description">A Workflow Id is a customizable, application-level identifier for a Workflow Execution that is unique to an Open Workflow Execution within a Namespace.</div><div class="preview-modal-tags"><a class="preview-modal-tag" href="/tags/term">term</a> <a class="preview-modal-tag" href="/tags/explanation">explanation</a></div></div> to function.

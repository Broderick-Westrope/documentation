## backfill

The `temporal schedule backfill` command executes Actions ahead of their specified time range.
Backfilling can be used to fill in [Workflow Runs](/workflows#run-id) <span id="i-b43bf473-c382-439a-b202-be9df0aeaf8a" class="clickable-i clickable-link-preview">ðŸ”—</span><div id="preview-modal-b43bf473-c382-439a-b202-be9df0aeaf8a" class="preview-modal"><div class="modal-header"><div id="x-b43bf473-c382-439a-b202-be9df0aeaf8a" class="clickable-x clickable-link-preview">x</div><b>Link preview</b></div><div class="preview-modal-title">What is a Run Id?</div><div class="preview-modal-description">A Run Id is a globally unique, platform-level identifier for a Workflow Execution.</div><div class="preview-modal-tags"><span class="preview-modal-tag">term</span> <span class="preview-modal-tag">explanation</span></div></div> from a time period when the Schedule was paused, or from before the Schedule was created.

Schedule backfills require a valid Schedule ID, along with the time in which to run the Schedule and a change to the overlap policy.

```
temporal schedule backfill --schedule-id 'your-schedule-id' \
--overlap-policy 'BufferAll' 				\
--start-time '2022-05-0101T00:00:00Z'		\
--end-time '2022-05-31T23:59:59Z'
```

## create

The `temporal schedule create` command creates a new [Schedule](/workflows#schedule) <span id="i-4bfa8161-8194-487f-8114-8bf9232fc1a3" class="clickable-i clickable-link-preview">ðŸ”—</span><div id="preview-modal-4bfa8161-8194-487f-8114-8bf9232fc1a3" class="preview-modal"><div class="modal-header"><div id="x-4bfa8161-8194-487f-8114-8bf9232fc1a3" class="clickable-x clickable-link-preview">x</div><b>Link preview</b></div><div class="preview-modal-title">What is a Schedule?</div><div class="preview-modal-description">A Schedule enables the scheduling of Workflow Executions.</div><div class="preview-modal-tags"><span class="preview-modal-tag">term</span> <span class="preview-modal-tag">explanation</span></div></div>.
Newly created Schedules return a Schedule ID to be used in other Schedule commands.

Schedules need to follow a format like the example shown here:

```
temporal schedule create \
    --schedule-id 'your-schedule-id' \
    --workflow-id 'your-workflow-id' \
    --task-queue 'your-task-queue' \
    --workflow-type 'YourWorkflowType'
```

Any combination of `--calendar`, `--interval`, and `--cron` is supported.
Actions will be executed at any time specified in the Schedule.

## delete

The `temporal schedule delete` command deletes a [Schedule](/workflows#schedule) <span id="i-1b8b5d7f-91ee-4cea-a1f4-a4e068ead21d" class="clickable-i clickable-link-preview">ðŸ”—</span><div id="preview-modal-1b8b5d7f-91ee-4cea-a1f4-a4e068ead21d" class="preview-modal"><div class="modal-header"><div id="x-1b8b5d7f-91ee-4cea-a1f4-a4e068ead21d" class="clickable-x clickable-link-preview">x</div><b>Link preview</b></div><div class="preview-modal-title">What is a Schedule?</div><div class="preview-modal-description">A Schedule enables the scheduling of Workflow Executions.</div><div class="preview-modal-tags"><span class="preview-modal-tag">term</span> <span class="preview-modal-tag">explanation</span></div></div>.
Deleting a Schedule does not affect any [Workflows](/workflows#) <span id="i-d32635d7-4850-4487-a362-cdf485524c55" class="clickable-i clickable-link-preview">ðŸ”—</span><div id="preview-modal-d32635d7-4850-4487-a362-cdf485524c55" class="preview-modal"><div class="modal-header"><div id="x-d32635d7-4850-4487-a362-cdf485524c55" class="clickable-x clickable-link-preview">x</div><b>Link preview</b></div><div class="preview-modal-title">What is a Workflow?</div><div class="preview-modal-description">In day-to-day conversations, the term "Workflow" frequently denotes either a Workflow Type, a Workflow Definition, or a Workflow Execution.</div><div class="preview-modal-tags"><span class="preview-modal-tag">term</span> <span class="preview-modal-tag">explanation</span></div></div> started by the Schedule.

[Workflow Executions](/workflows#workflow-execution) <span id="i-c2bcd36d-71bb-4115-8d80-7275ef829fe6" class="clickable-i clickable-link-preview">ðŸ”—</span><div id="preview-modal-c2bcd36d-71bb-4115-8d80-7275ef829fe6" class="preview-modal"><div class="modal-header"><div id="x-c2bcd36d-71bb-4115-8d80-7275ef829fe6" class="clickable-x clickable-link-preview">x</div><b>Link preview</b></div><div class="preview-modal-title">What is a Workflow Execution?</div><div class="preview-modal-description">A Temporal Workflow Execution is a durable, scalable, reliable, and reactive function execution. It is the main unit of execution of a Temporal Application.</div><div class="preview-modal-tags"><span class="preview-modal-tag">term</span> <span class="preview-modal-tag">explanation</span></div></div> started by Schedules can be cancelled or terminated like other Workflow Executions.
However, Workflow Executions started by a Schedule can be identified by their [Search Attributes](/visibility#search-attribute) <span id="i-9a5ede4d-a604-44b0-940e-4d5f43dae903" class="clickable-i clickable-link-preview">ðŸ”—</span><div id="preview-modal-9a5ede4d-a604-44b0-940e-4d5f43dae903" class="preview-modal"><div class="modal-header"><div id="x-9a5ede4d-a604-44b0-940e-4d5f43dae903" class="clickable-x clickable-link-preview">x</div><b>Link preview</b></div><div class="preview-modal-title">What is a Search Attribute?</div><div class="preview-modal-description">A Search Attribute is an indexed name used in List Filters to filter a list of Workflow Executions that have the Search Attribute in their metadata.</div><div class="preview-modal-tags"><span class="preview-modal-tag">term</span> <span class="preview-modal-tag">explanation</span> <span class="preview-modal-tag">filtered-lists</span> <span class="preview-modal-tag">visibility</span></div></div>, making them targetable by batch command for termination.
